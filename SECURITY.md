# Security Policy / セキュリティポリシー

## Supported Versions / サポートされているバージョン

Use this section to tell people about which versions of your project are currently being supported with security updates.
現在セキュリティアップデートでサポートされているプロジェクトのバージョンは以下の通りです。

| Version / バージョン | Supported / サポート |
| ------------------- | ------------------ |
| 0.1.x               | :white_check_mark: |
| < 0.1               | :x:                |

## Reporting a Vulnerability / 脆弱性の報告方法

If you find a security vulnerability, please report it by opening an issue on GitHub. We will review the report and provide an update as soon as possible.
脆弱性を発見した場合は、GitHubのIssueを作成して報告してください。報告内容を確認し、可能な限り速やかにアップデートを提供します。

Once a vulnerability is reported, you can expect an initial response within 48 hours. If the vulnerability is accepted, we will work on a fix and release a new version. If it is declined, we will provide a reason why.
報告を受けた後、48時間以内に初期回答を行うよう努めます。脆弱性が承認された場合は、修正作業を行い、新しいバージョンをリリースします。拒否された場合は、その理由を説明します。
